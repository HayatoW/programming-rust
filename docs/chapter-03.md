# 3章 基本的な型

全ての型を明示的に書く

```rust
fn build_vector() -> Vec<i16> {
    let mut v: Vec<i16> = Vec::<v16>::new();
    v.push(10i16);
    v.push(20i16);
    v
}
```

型推論のおかげで次のように書けばいい

```rust
fn build_vector() -> Vec<i16> {
    let mut v = Vec::<v16>::new();
    v.push(10i16);
    v.push(20i16);
    v
}
```

Rust のジェネリック関数の効率はジェネリックでない関数と変わらない。

| 型 | 説明 | 値の例 |
| --- | --- | --- |
| Box<Attend> | ボックス: ヒープ上の値へのポインタを保持する | Box::new(Late(15)) |
| &dyn Any, &mut dyn Read | トレイトオブジェクト: 指定されたメソッドの集合を実装した任意の値への参照。 | value as &dyn Any, &mut file as &mut dyn Read |

Rust ではバイト値に対して u8 型を用いる。

Rust では配列のインデックスは usize にしなければならない。

stand-in 記法: `b'\t'` (バイトリテラルでタブを表現)

メソッド呼び出しの優先順位は単項前置演算子より高い

デバッグビルドでは、整数演算中のオーバーフローは検出され、パニックを発生させる。<br>
リリースビルドでは、wrap されるので、1周して別ビットパターンになったりする。

Rust は独立した1文字に対しては char を用いるが、文字列やテキストのストリームに対しては UTF-8 エンコードを用いる。つまり String はテキストを UTF-8 バイト列として保持するのであって、文字の配列文字の配列を保持するわけではない。

char 型の文字リテラルは、'8' や '!' のようにシングルクォートで囲む。

Rust では、as 演算子は、安価で失敗することのない変換のみに限定している。

タプルでは個々の要素が異なる型を持つことができるのに対して、配列では全ての要素が同じ型でなければならない。

```rust
// この宣言は以下の宣言の省略記法
fn swap<T>(x: &mut T, y: &mut T);

// ユニット型 (0要素のタプル) を返す
fn swap<T>(x: &mut T, y: &mut T) -> ();
```

Java では、例えば class Rectangle にフィールド Vector2D upperLeft があるとすると、upperLeft は別に作られた Vector2D オブジェクトへの参照になる。Java ではあるオブジェクトに別のオブジェクトが物理的に含まれることはない。<br>
Rust は違う。Rust はメモリ消費を最小にするように設計されているので、値はデフォルトでネストする。

その結果として、Rust ではある値から別の値を指す場合には、必ず明示的にポインタを用いなければならなくなっている。
