| トレイト | 説明 |
| --- | --- |
| Drop | デストラクタ。値がドロップされる際に自動的に実行される後始末コード |
| Sized | コンパイル時にサイズが決まる型に対するマーカートレイト。スライスなどの動的にサイズが決まる型ではないことを示す |
| Clone | 値をクローン可能な型 |
| Copy | 値の置かれたメモリをバイト単位でコピーすることでクローン可能であるということを示すマーカートレイト |
| Deref<br>DerefMut | スマートポインタ型のためのトレイト |
| Default | 実用的な「デフォルト値」を持つ型であることを示す |
| AsRef<br>AsMut | ある参照型から別の参照型へ変換するためのトレイと |
| Borrow<br>BorrowMut | AsRef/AsMut と同様の変換トレイトだが、ハッシュ値が同じであること、順序、等価性が同じであることを保証する |
| From<br>Into | ある型から別の型への変換トレイト |
| TryFrom<br>TryInto | ある型から別の型への変換トレイト。失敗する可能性のある変換に用いる |
| ToOwned | 参照から所有されている値への変換トレイト |

一般に、可能な限り clone_from を使って、最適化されたコードが提供されていればそれが使われるようにするべき

適切な From の実装があれば、標準ライブラリが自動的に対応する Into トレイトを実装してくれる。<br>
独自の型を実装する際に、引数が1つだけのコンストラクタを作るのなら、対応する型からの `From<T>` の実装として作成するとよい。<br>
そうすれば、対応する Into の実装が無料で手に入るからだ。
